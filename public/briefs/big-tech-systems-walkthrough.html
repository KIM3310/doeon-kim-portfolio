5-minute path

The highest-signal sequence

1. StagePilot

Use runtime brief, perf evidence, and failure taxonomy to show bounded API posture and honest claim boundaries.

2. AegisOps

Use the system design pack and postmortem pack to show topology, hot paths, review surfaces, and commander handoff.

3. TwinCity UI / Enterprise Kit

Show that the same engineering judgment survives in stateful operator UX and governed rollout workflows.

What to emphasize

The design topics that matter most

Service clarity

Explain what each API promises, what it does not promise, and where human review still matters.

Operational honesty

Call out where the public package is benchmark-backed and where it is reviewer-backed rather than live internet scale.

Failure drills

Questions worth preparing for

Be ready to walk through a concrete failure: how it shows up, where it is bounded, when it escalates, and how a human operator re-enters the loop. That is the strongest systems signal in this portfolio.
