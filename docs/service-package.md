# Service Package - doeon-kim-portfolio

This package defines the most practical service shape for this repository. It is public-safe: no price promises, financial projections, or buyer-specific contract assumptions are published here.

## Buyer and Wedge

- **Service lane:** Portfolio service router
- **Primary buyer:** Founders, enterprise AI leaders, technical buyers, talent partners, and technical reviewers
- **Front-door offer:** Service package discovery surface
- **Service outcome:** Turn the whole portfolio into a clear menu of proof-led service packages and contact paths.

## Build and Polish Priorities

- Keep service packages visible without clutter
- Hide dense proof behind progressive disclosure
- Keep public copy free of projections and price promises

## Buyer-Ready Deliverables

- Service package matrix
- Live proof gallery
- Private intake handoff

## Activation Path

1. Use the public demo, README, screenshots, and tests to establish proof.
2. Confirm the buyer workflow, data boundary, owner, and acceptance criteria.
3. Build only the smallest workflow needed to prove the service outcome.
4. Add cloud resources, accounts, storage, or paid integrations only after the workflow boundary is approved.

## Margin and Risk Controls

- Keep the static site fast and add CRM/calendar/payment only after inbound volume justifies it.
- Keep the first service scope narrow, repeatable, and evidence-led.
- Keep private financial planning outside the public repository.
- Keep logs, analytics, and reports free of private payloads unless explicit approval exists.

## Portfolio Signal

The portfolio should present this repository as **Service package discovery surface** under the **Portfolio service router** lane, with proof links routed to the live demo or repository evidence.
