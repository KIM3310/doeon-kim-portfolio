## Shared Free-Tier Stack

- **Front door:** Cloudflare Pages first, with GitHub Pages, Vercel, or Netlify when repository shape fits better.
- **Serverless/API:** Cloudflare Workers for thin APIs and metering gates; Supabase or Firebase when managed auth and data are needed.
- **Storage:** R2, Supabase Storage, Firebase Storage, or repository artifacts depending on privacy and artifact size.
- **AI inference:** OpenRouter, Groq, Cerebras, Cloudflare Workers AI, NVIDIA NIM, and Ollama/customer-key fallback for heavier or private workloads.
- **ML/demo hosting:** Hugging Face Spaces for public CPU demos; local/customer-owned runtime for private data.
- **Development loop:** OpenCode, Kimi Code CLI, Freebuff, Lovable, and local Ollama-assisted agents for fast iteration without committing to paid SaaS.

## Revenue Architecture Rule

Do not build a paid platform first. Ship a static or serverless public surface, add a lead or export action, instrument quota/entitlement fields, then sell one fixed-scope paid package. Only after repeated demand should a repository take on always-on infrastructure, paid model usage, or subscription complexity.
